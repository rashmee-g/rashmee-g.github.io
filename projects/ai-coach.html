AI Public Speaking Coach

A mobile app that gives real-time and post-session feedback on delivery, body language, facial expressions, and content focus using multimodal AI.

React Native Computer Vision Audio Analysis NLP Node.js + Python
How It Works View Documentation

My Role

Frontend UI lead: user flows, feedback visualizations, and progress tracking. I collaborate with the team to integrate the analysis pipeline and test modules.

Team

5-person capstone team working in milestones: design, module development, integration, testing, and final presentation.

Problem

Many people practice speeches alone and don’t get consistent, objective feedback. Traditional coaching can be limited by cost, availability, and subjectivity, and many existing apps only focus on one aspect (like pace or filler words).

Solution

The app captures audio and video during practice, then combines vision, audio, and language analysis to deliver: quick, non-intrusive real-time cues and a detailed post-session summary with strengths, improvement areas, and trends over time.

Real-Time Cues

High-level nudges (example: pacing or posture) designed to avoid overwhelming the speaker.

Session Report

Scores, examples, and actionable suggestions across voice, body language, facial signals, and focus.

Progress Tracking

Trends over repeated sessions so users can measure improvement and build confidence.

Scope

This project focuses on a practical coaching tool (not a replacement for professional training). It evaluates four core dimensions: body language, facial expressions, speech delivery, and content relevance, with controlled, privacy-aware practice sessions.

In Scope

  • Body language: posture, gestures, movement, eye contact
  • Facial expressions: engagement/confidence indicators
  • Speech delivery: pace, pitch variation, pauses, filler words
  • Content relevance: topic drift using transcript vs outline

Out of Scope

  • Live audience interaction
  • Multilingual analysis (initial versions)
  • Advanced emotional coaching beyond basic indicators

How It Works

The app records a practice session using the phone camera and microphone. A backend service coordinates vision, audio, and NLP modules in parallel, then merges results into a unified feedback report.

Core Modules

  • Vision: pose and gesture signals
  • Face: engagement/confidence indicators
  • Audio: pacing, pitch variation, fillers
  • NLP: transcript alignment + topic drift

Tech Stack

  • Frontend: React Native
  • Backend: Node.js + Python services
  • APIs: REST endpoints between services
  • Models/tools: MMPose, DeepFace, Librosa, Speech-to-Text, Sentence-BERT

Testing and Validation Plan

We plan to unit test each module, run integration tests across the pipeline, and validate the end-to-end experience in controlled practice sessions. We’ll track reliability and responsiveness so the feedback feels immediate and consistent.

Testing

  • Unit tests for each analysis module
  • Integration testing across multimodal pipeline
  • User-based validation for usability and accuracy

Metrics

  • Speech rate accuracy
  • Filler word detection reliability
  • System responsiveness and latency

Project Roadmap

Work is organized into milestones: requirements and design, module development, multimodal integration, user testing, and final evaluation with presentation.

Phase 1

Requirements and system design, architecture planning, early UI wireframes.

Phase 2–3

Module development and testing, then integration into a unified feedback pipeline.

Phase 4–5

User testing, refinement, final evaluation, documentation, and capstone presentation.
