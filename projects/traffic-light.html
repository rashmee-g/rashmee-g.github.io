Hardware-Integrated Traffic Light System

A sequential logic circuit that simulates real traffic signal timing and transitions, designed and validated through iterative debugging.

Sequential Logic Timing Design Hardware Debugging Digital Circuits
See The Build Design Process Reflection

Goal

Create a reliable traffic light cycle (green → yellow → red) with correct timing behavior and clean state transitions.

What I Focused On

Translating the state sequence into hardware logic, validating timing, and debugging until transitions behaved consistently.

Overview

This project models a real-world control system using a finite set of states and timed transitions. The core challenge was ensuring the circuit consistently followed the intended sequence and didn’t glitch during transitions.

What I Built

The final system runs a complete cycle and drives the correct light outputs for each state.

  • Designed the state sequence and output behavior for each light phase.
  • Implemented timing + transitions using sequential logic and validated signal stability.
  • Debugged issues like incorrect wiring, inconsistent state changes, and timing drift until the cycle was reliable.
Final hardware build: tested end-to-end to confirm consistent light transitions.
Overall setup during testing.

Design Process

I approached this like a hardware state machine: define states, map outputs, then validate transitions under real timing conditions.

1) Define states + outputs

  • Set states for each phase (ex: Green, Yellow, Red)
  • Assigned outputs to ensure only the intended light is active per state
  • Verified outputs don’t conflict during transitions

2) Validate timing + transitions

  • Tested that each phase holds for the correct duration
  • Checked transitions for stability (no flicker / unexpected states)
  • Iterated until behavior matched the intended cycle
